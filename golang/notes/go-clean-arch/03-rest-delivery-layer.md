# go-clean-arch · Delivery 层拆解（internal/rest/article.go）

> bxcodec/go-clean-arch v4 中"对外暴露"的那一层。本笔记接续 [02 · main.go 拆解](./02-main-go-clean-arch.md)，看 HTTP 接口是怎么注册、被调用、把领域错误翻译成 HTTP 状态码的。

## 核心要点

- **接口在消费方声明**：`ArticleService` 接口声明在 `internal/rest/article.go` 里，由 `article.Service` 隐式实现。这个反向声明让 mockery 一键生成 `mocks/ArticleService.go`，handler 测试不需要拉起 MySQL。
- **`NewArticleHandler` 是副作用构造**：不接受返回值，只把路由注册到传入的 `*echo.Echo` 实例上。main.go 只调一次，返回值直接丢。
- **路由表 ≠ 接口表**：`ArticleService` 接口有 6 个方法，但只注册了 4 条路由（`Update` 和 `GetByTitle` 缺）。这是有意取舍，未来加 `PATCH /articles/:id` 即可补齐。
- **业务错误到 HTTP 状态的翻译**：`getStatusCode(err)` 把 `domain.ErrNotFound` 等 sentinel error 翻译成 404/409/500。domain 层不引用 `net/http`，将来加 gRPC delivery 时各译各的。

## 文件骨架

```
package rest

type ArticleService interface { ... }                          // ① 契约（消费方声明）
type ArticleHandler struct { Service ArticleService }          // ② handler

func NewArticleHandler(e *echo.Echo, svc ArticleService)        // ③ 构造 + 注册
func (a *ArticleHandler) FetchArticle(c echo.Context) error    // ④ GET /articles
func (a *ArticleHandler) GetByID(c echo.Context) error         // ⑤ GET /articles/:id
func (a *ArticleHandler) Store(c echo.Context) error           // ⑥ POST /articles
func (a *ArticleHandler) Delete(c echo.Context) error          // ⑦ DELETE /articles/:id

func isRequestValid(m *domain.Article) (bool, error)           // ⑧ 校验（go-playground/validator）
func getStatusCode(err error) int                              // ⑨ 错误翻译
```

## 关键示例

### 接口在消费方声明 + mockery 自动生成 mock

```go
// internal/rest/article.go (delivery 侧声明它需要的)
type ArticleService interface {
    Fetch(ctx, cursor string, num int64) ([]domain.Article, string, error)
    GetByID(ctx, id int64) (domain.Article, error)
    Update(ctx, ar *domain.Article) error
    GetByTitle(ctx, title string) (domain.Article, error)
    Store(context.Context, *domain.Article) error
    Delete(ctx, id int64) error
}
```

`article/service.go` 顶部有一行：

```go
//go:generate mockery --name ArticleService
```

跑 `go generate ./...` 后，`internal/rest/mocks/ArticleService.go` 自动生成（顶部写着 `Code generated by mockery v2.42.0. DO NOT EDIT.`），handler 测试就可以这样：

```go
mockUCase := new(mocks.ArticleService)
mockUCase.On("Fetch", mock.Anything, cursor, int64(num)).
    Return(mockListArticle, "10", nil)

handler := rest.ArticleHandler{ Service: mockUCase }
handler.FetchArticle(c)  // 完全不碰 MySQL
```

### 副作用构造函数：构造 + 路由注册一次完成

```go
func NewArticleHandler(e *echo.Echo, svc ArticleService) {
    handler := &ArticleHandler{ Service: svc }
    e.GET("/articles",         handler.FetchArticle)
    e.POST("/articles",        handler.Store)
    e.GET("/articles/:id",     handler.GetByID)
    e.DELETE("/articles/:id",  handler.Delete)
    // 🚨 Update 没有注册路由（接口里有，handler 里没有）
}
```

调用一次即可：

```go
// app/main.go
rest.NewArticleHandler(e, svc)   // 返回值丢弃
```

### 端点方法的标准动作

| 操作 | Echo API | 备注 |
|---|---|---|
| 读 query | `c.QueryParam("num")` | `?num=20&cursor=xxx` |
| 读 path | `c.Param("id")` | `/articles/:id` |
| 读 body | `c.Bind(&article)` | 自动按 `Content-Type` 反序列化 |
| 写响应 | `return c.JSON(http.StatusOK, body)` | Echo 推荐写法：error 也是 return |
| 写 header | `c.Response().Header().Set("X-Cursor", nextCursor)` | 游标放 header 比放 body 更 RESTful |
| 拿 ctx | `c.Request().Context()` | 带 timeout 的 ctx，从 middleware 透传 |

### 错误翻译：domain 不知道 HTTP 存在

```go
func getStatusCode(err error) int {
    if err == nil { return http.StatusOK }
    logrus.Error(err)
    switch err {
    case domain.ErrInternalServerError: return http.StatusInternalServerError
    case domain.ErrNotFound:            return http.StatusNotFound
    case domain.ErrConflict:           return http.StatusConflict
    default:                           return http.StatusInternalServerError
    }
}
```

调用处：

```go
if err != nil {
    return c.JSON(getStatusCode(err), ResponseError{Message: err.Error()})
}
```

## 路由与接口对照表（已知设计取舍）

| 方法 | 接口里有 | 路由里有 | 备注 |
|---|:---:|:---:|---|
| Fetch | ✅ | ✅ `GET /articles` | 列表 + 游标分页 |
| GetByID | ✅ | ✅ `GET /articles/:id` | 路径参数 |
| GetByTitle | ✅ | ❌ | 没暴露 |
| **Update** | ✅ | **❌ 没注册** | 未来加 `PATCH /articles/:id` |
| Store | ✅ | ✅ `POST /articles` | 含 `validate:"required"` 校验 |
| Delete | ✅ | ✅ `DELETE /articles/:id` | 返回 204 |

## 常见坑

- **`switch err ==` 只对未 wrap 的 sentinel error 生效**：`domain/errors.go` 用 `errors.New(...)`，所以能用 `==`；一旦重构为 `fmt.Errorf("%w: not found", ErrInternalServerError)` 这种 wrap 形式，就必须改用 `errors.Is`，否则全走到 `default` 分支返 500。
- **`GetByID` 把 `Atoi` 失败返 404 而非 400**：路径参数非法是客户端语法错误，正确状态应是 `Bad Request`。
- **`X-Cursor` 没设 `c.Header(...)`，用了 `c.Response().Header()`**：两者都能写，但前者更"Echo-native"，并保证在后面 `c.JSON()` 之前生效。
- **`Update` 接口有但没注册**：依赖外部契约的实现必须满足接口；将来从 `service.go` 删掉 `Update` 之前要先确认没有调用方。路由表同步未跟上是 v4 已知瑕疵。
- **`Store` 校验后没回写 ID**：service/Repository 已经做了，但 handler 没做特殊处理；多数项目会在这里 `article.ID = lastID` 后再返 201。

## 配套练习

- ⭐ 加 `Update` 路由：`e.PATCH("/articles/:id", handler.Update)` + 写 `Update(c echo.Context) error`，复用 `c.Bind` + `isRequestValid`。
- ⭐⭐ 把 `domain/errors.go` 改成 wrap 形式：`ErrNotFound = fmt.Errorf("%w: not found", ErrInternalServerError)`，再把 `getStatusCode` 改成 `errors.Is` 链。
- ⭐⭐⭐ 把 `internal/rest` 拆成独立 module，让 `article` 包不再依赖 `echo`（只在 main.go 注册时导入 echo）。这是 v5 候选方向。
